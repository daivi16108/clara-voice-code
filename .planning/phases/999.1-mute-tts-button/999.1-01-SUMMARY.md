# Phase 999.1 Plan 01: Mute TTS Button Summary

**Ephemeral TTS mute toggle: webview speaker button syncs tts_muted to shared settings, MCP voice_speak skips audio and returns text-only notification when muted**

- **Completed:** 2026-03-29T11:29:35Z

- **Files modified:** 3 (+ 1 created)

## Accomplishments
- Added mute/unmute speaker button in PTT webview panel next to hold-to-talk button
- MCP voice_speak checks tts_muted in shared settings — skips speak.py spawn, returns explicit "TTS muted — text only mode" response with content preview
- voice_ask also skips speaking the question when muted (still listens for voice answer)
- voice_status includes tts_muted field so Claude can query current state
- 5 unit tests covering all mute/unmute scenarios pass

## Files Created/Modified

- `src/mcp/standalone.ts` - Added isTtsMuted(), mute check in toolSpeak/toolAsk, tts_muted in toolStatus
- `src/extension.ts` - Added ttsMuted flag, syncMuteState(), mute button in webview HTML, tts_mute_toggle postMessage handler, i18n mute/unmute strings, status bar mute indicator

## Decisions Made
- Mute state is ephemeral (`let ttsMuted = false`) — not persisted to settings (D-06, D-07 from context)
- voice_speak returns content preview alongside muted notification so Claude knows what was not spoken
- voice_speak tool stays visible/callable when muted — Claude decides whether to call it (D-05)
- Mute button uses Unicode emoji: U+1F50A (speaker with sound) and U+1F507 (speaker off/muted)

None — plan executed exactly as written.

These failures were confirmed pre-existing by stash verification and are out of scope per deviation scope boundary rules. Logged to deferred-items.
