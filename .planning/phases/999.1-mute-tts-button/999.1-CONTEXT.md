# Phase 999.1: Mute TTS Button - Context

**Gathered:** 2026-03-29
**Status:** Ready for planning

<domain>
## Phase Boundary

Add a mute TTS toggle button to the PTT webview panel. When muted, voice_speak skips audio playback and returns an explicit "muted" response so Claude knows to respond with text only. Mute state is ephemeral (in-memory, resets on VS Code restart).

</domain>

<decisions>
## Implementation Decisions

### Button Placement
- **D-01:** Mute button goes in the PTT webview panel, next to the existing microphone/record button
- **D-02:** Not in status bar, not in Command Palette only — all voice controls in one place

### voice_speak Behavior When Muted
- **D-03:** voice_speak returns explicit notification: `"TTS muted — text only mode. Your message was not spoken."` with `{ success: true, muted: true }`
- **D-04:** Claude sees the muted response and can adapt (write longer text instead of short voice phrases)
- **D-05:** voice_speak tool remains registered even when muted — it's not hidden from Claude

### State Management
- **D-06:** Mute state is an in-memory flag, not persisted to VS Code settings
- **D-07:** On VS Code restart, mute is always OFF (voice enabled by default)
- **D-08:** No `claraVoice.ttsMuted` setting — this is intentionally ephemeral

### Claude's Discretion
- Toggle icon design (muted/unmuted visual states)
- How the mute flag is communicated between extension and MCP server (shared settings file, env var, or IPC)
- Whether voice_status also reflects mute state (recommended: yes)

</decisions>
